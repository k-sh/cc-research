# Phase 7: Release

研究成果(コード・モデル・データ)を公開できる状態に整えるフェーズです。

## research-14-release-preparation

| 項目 | 内容 |
| --- | --- |
| コマンド | `/research-14-release-preparation` |
| 使用モデル | Haiku |
| 目的 | GitHub公開、README、モデルカード、ライセンス、リリース資料を整備する |

### いつ使うか

- 「GitHubで公開したい」「OSSとしてリリースしたい」「READMEを整備したい」
  「モデルカードを作りたい」「公開準備」

### 主な処理

1. **公開前の安全確認**: シークレット・個人情報・社外秘データの混入がないか、依存関係のライセンスが
   公開と両立するかを確認する。問題があれば作業を止めて報告する。
2. **ライセンス選定**: 候補(MIT / Apache-2.0 / BSD-3 / GPL 系 / CC 系)をユーザーに提示して選定する。
3. **ドキュメント整備**: README(概要・インストール・実行例・主要結果・引用方法)、モデルカード、
   データカード、CITATION.cff、リリースノートを作成する。
4. **公開チェックリスト**: シークレットスキャン・ライセンス整合・手順の動作確認・不要ファイルの
   除去などをチェックリスト化する。

### 出力

```
release/
    README.md            # 概要、インストール、実行例、主要結果、引用方法、ライセンス
    MODEL_CARD.md         # モデル概要、学習データ、性能、想定用途、限界・バイアス
    DATA_CARD.md          # データ概要、収集方法、ライセンス、既知の偏り
    CITATION.cff          # 引用情報
    RELEASE_NOTES.md      # バージョン、含まれるもの、既知の問題
    release-checklist.md  # 公開チェックリスト
```

### 品質基準

- 公開は不可逆であることを踏まえ、安全確認を省略しない
- README の Quickstart は必ず動作確認した手順にする
- 数値・主張は `paper.md` / `evaluation-report.md` からの転記とし、新しい主張を作らない
