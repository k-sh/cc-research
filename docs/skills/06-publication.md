# Phase 6: Publication

研究成果を論文化し、発表資料を作成し、査読対応を行うフェーズです。

## research-11-paper-writing

| 項目 | 内容 |
| --- | --- |
| コマンド | `/research-11-paper-writing` |
| 使用モデル | Opus |
| 目的 | 研究成果を論文(Markdown)としてまとめる |

### いつ使うか

- 「論文を書きたい」「論文化したい」「原稿をまとめたい」「アブストを書きたい」
- `research-10-reproducibility` の次のステップとして

### 執筆の原則

- **すべての主張に証拠を対応させる**: 性能の主張は `experiment-results.md` の数値に、
  新規性の主張は `research-gap.md` の分析に遡れること
- **引用は検証済みのものだけ**: `related-works.md` に URL/DOI 付きで記録された文献のみを引用する。
  記憶から参考文献を生成しない
- **記法の一貫性**: `algorithm-design.md` の記法表に従う

### 主な処理

1. 論文の中核主張と各セクションのアウトラインをユーザーと合意する。
2. Method → Experiments → Results → Related Work → Introduction → Discussion → Conclusion →
   Abstract の順で執筆する(Abstract とタイトルは最後)。
3. セルフレビュー: 貢献の箇条書きと Results の証拠の対応、引用の実在、数値の一致、記法のゆれを
   確認する。

### 出力

```
paper/
    paper.md    # 論文本体(References、Appendix含む)
```

## research-12-slide-generation

| 項目 | 内容 |
| --- | --- |
| コマンド | `/research-12-slide-generation [発表時間・発表先]` |
| 使用モデル | Sonnet |
| 目的 | 論文から発表資料・発表原稿・想定Q&Aを作成する |

### いつ使うか

- 「発表資料を作りたい」「スライドを作りたい」「学会発表の準備」「想定質問を用意したい」
- `research-11-paper-writing` の次のステップとして

### 主な処理

1. **構成設計**: 発表時間から逆算して枚数・時間配分を決める(目安 1 枚 1〜1.5 分)。
2. **スライド本文**: Marp 互換 Markdown で、1 スライド 1 メッセージの原則に沿って作成する。
3. **発表原稿**: スライドごとに想定秒数付きで台本を書く。
4. **想定 Q&A**: 限界・実験設定の妥当性・不採用にした代替案などの観点から最低 10 問用意する。

### 出力

```
presentation/
    slides.md   # Marp互換スライド
    speech.md   # 発表原稿(秒数付き)
    qa.md       # 想定Q&A
```

## research-13-review-response

| 項目 | 内容 |
| --- | --- |
| コマンド | `/research-13-review-response [査読コメント]` |
| 使用モデル | Sonnet |
| 目的 | 査読コメントへの回答・修正履歴・再投稿資料を作成する |

### いつ使うか

- 「査読コメントが来た」「レビューに回答したい」「リバッタルを書きたい」「再投稿の準備」

### 主な処理

1. **コメントの分解と分類**: 査読者・コメントごとに ID を振り、種別(誤解/妥当な指摘/追加実験要求/
   意見の相違/軽微)と重大度、対応方針を整理する。
2. **回答の作成**: 感謝 → 回答 → 変更箇所の順で書く。反論する場合は証拠を添えて敬意を保つ。
3. **論文の修正と履歴**: 対応方針に沿って論文を修正し、修正履歴を記録する。
4. **再投稿前チェック**: 全コメントへの回答漏れ、回答と修正の整合性を確認する。

### 出力

```
paper/
    review-response.md    # コメント分類表 + Response Letter
    revision-history.md   # 修正履歴
    paper.md              # 改訂稿(更新)
```

### 品質基準(3スキル共通)

- 過大な主張をせず、実験で示した範囲を超えて一般化しない
- 限界を隠さない(査読対応で自分を守る材料になる)
- 「対応した振り」をしない: 回答に書いたことは必ず本文に反映する
