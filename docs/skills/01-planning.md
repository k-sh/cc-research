# Phase 1: Planning

研究テーマを明確化し、以降の全工程の土台となる研究計画を作成するフェーズです。

## research-01-planning

| 項目 | 内容 |
| --- | --- |
| コマンド | `/research-01-planning [研究テーマの概要]` |
| 使用モデル | Sonnet |
| 目的 | 研究目的・制約・評価指標・調査キーワード・調査計画を整理する |

### いつ使うか

- 新しい研究テーマに着手するとき(パイプラインの起点)
- 「研究を始めたい」「研究計画を立てたい」「テーマを整理したい」と言ったとき

### 主な処理

1. **ヒアリング**: 研究目的(抽象化した本当の目的)、研究分野、制約(期間・計算資源・データ・予算)、
   実験環境、評価指標、想定成果物、スコープを AskUserQuestion で確認する。すでに分かっている項目は
   聞き直さない。
2. **文書化**: 目的・制約・評価指標・研究計画を Markdown にまとめる。研究計画には次回以降で使う
   調査キーワード(日英両方)とフェーズ計画、リスクと代替案を含める。
3. **STATUS.md の初期化**: 14 フェーズ分の進行状況を管理するファイルを作成する。

### 出力

```
research/
    objective.md           # 研究目的(背景・動機・目的の抽象化・成功基準)
    constraints.md         # 制約事項
    evaluation-metrics.md  # 評価指標(定義・測定方法・目標値)
    research-plan.md       # 研究計画(スコープ・調査キーワード・フェーズ計画・リスク)
    STATUS.md              # 進行状況トラッカー(以降全フェーズが更新)
```

### 品質基準

- 成功基準は測定可能な形で記述する(例:「データセット X で指標 Y をベースライン比 Z% 改善」)
- 自分が置いた仮定はユーザー発言と区別して明示する
- 未確定の制約も「未確定」として記録し、後工程の判断材料として残す
