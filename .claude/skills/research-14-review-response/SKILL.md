# 査読対応(Review Response)

査読コメントを整理し、Response Letter・修正履歴・再投稿資料を作成する。

## 前提

開始時に `research/STATUS.md`・`paper/paper.md` と、ユーザーから受け取った査読コメント(引数のパスまたは貼り付け)を読む。
コメントが未提供なら最初に依頼する。締切と再投稿規定(ページ制限・修正箇所の明示方法)も確認する。

## 進め方

### 1. コメントの分解と分類

査読者ごと・コメントごとに ID を振り(R1-C1, R1-C2, ...)、分類する:

- **種別**: 誤解(説明不足) / 妥当な指摘(修正必要) / 追加実験要求 / 意見の相違 / 軽微(誤字等)
- **重大度**: 採否に関わる / 改善要求 / 軽微
- **対応方針**: 修正する / 追加実験する / 丁寧に反論する / 一部受け入れて限定的に修正

全コメントの分類表を作り、対応方針をユーザーと合意してから回答を書く。

### 2. 回答の作成

各コメントへの回答の原則:

- **感謝 → 回答 → 変更箇所**の順。「We thank the reviewer... / ご指摘に感謝します」で始め、修正した場合は改訂稿の該当箇所(セクション・ページ)を明示する
- **誤解への対応**: 査読者を責めない。「説明が不十分でした」と受け、本文側を明確化する(回答文だけで説明して本文を直さない、をしない)
- **反論する場合**: 証拠(experiment-results.md・文献)を添えて敬意を保って反論する。感情的・防御的な表現を避ける
- **追加実験**: 実施可否を工数と締切から判断。できない場合は理由と代替(議論の追加、limitation への明記)を示す
- 根拠が必要な回答には research/ 配下の該当データ・文献を参照させ、記憶で数値を書かない

### 3. 論文の修正と履歴

- 対応方針に沿って `paper/paper.md` を修正する
- 修正箇所は `paper/revision-history.md` に記録: コメント ID / 修正前 → 修正後の要点 / セクション

### 4. 再投稿前チェック

- 全コメントに回答があるか(取りこぼしゼロ)
- 回答と実際の修正が一致しているか
- 修正によって他のセクションと矛盾が生じていないか(数値・主張の整合)
- 再投稿規定(ハイライト方法、分量)を満たしているか

### 5. 出力・完了処理

```
paper/
    review-response.md    # コメント分類表 + Response Letter(コメント→回答→変更箇所)
    revision-history.md   # 修正履歴
    paper.md              # 改訂稿(更新)
```

- `research/STATUS.md` を更新し、採択後は `/research-15-release-preparation` を案内

## 品質基準

- 一番厳しい査読者のコメントほど丁寧に扱う(採否を左右するのはその人)
- 「対応した振り」をしない: 回答に書いたことは必ず本文に反映されていること
