# 特許・知財調査

採用手法について特許面の先行技術を調査し、新規性と侵害リスクを評価する。

> **免責**: この調査は参考情報であり法的助言ではない。事業判断・出願判断の前には必ず弁理士等の専門家に確認するよう、成果物とユーザーへの報告の両方に明記すること。

## 前提

開始時に `research/STATUS.md`・`research/selected-method.md` を読む。無い場合は調査対象の技術内容をユーザーに質問する。

## 進め方

### 1. 検索クエリの設計

採用手法を構成要素に分解し、要素ごとに検索キーワードを作る(日英両方)。
技術用語の同義語・上位概念も含める(特許は独特の用語で書かれることが多い)。

### 2. 特許検索

WebSearch / WebFetch で以下を調査する:

- **Google Patents** (`patents.google.com`): まずここで広く検索
- **J-PlatPat**: 日本特許。日本語キーワードで検索
- **USPTO / Espacenet**: 米国・欧州特許

検索のコツ:
- CPC/IPC 分類コードが分かれば分類でも検索する
- 有力特許が見つかったら、その引用・被引用特許にも展開する
- 出願人(競合企業・大学)からの検索も行う

### 3. 類似特許の分析

類似度の高い特許(目安 3〜10 件)について記録する:

- 公開番号 / 出願人 / 出願日・公開日 / 法的状態(存続・失効・審査中、確認できる範囲で)
- 独立クレームの要旨
- **クレーム構成要素の対比表**: 特許のクレーム要素 vs 自分の手法の対応要素。全要素が一致しない限り文言侵害にはならない、という観点で要素ごとに ○/△/× を付ける

### 4. 評価と出力

```
research/
    patent-review.md      # 調査方法、検索クエリ、ヒット概況、新規性評価、侵害リスク評価、免責事項
    patent-comparison.md  # 類似特許ごとの詳細 + クレーム対比表 + 差別化案
```

- **新規性評価**: 手法の中核アイデアが既存特許のクレーム・明細書に開示されているか
- **侵害リスク評価**: 特許ごとに 高/中/低 + 理由。リスクがある場合は回避のための設計変更案(差別化案)を提示
- リスク「高」があれば method-design への差し戻しを提案する

### 5. 完了処理

- `research/STATUS.md` を更新(リスク総括を1行で記録)
- 次のステップ `/research-06-algorithm-design` を案内

## 品質基準

- クレーム(権利範囲)と明細書(開示内容)を区別して読む。新規性は明細書全体、侵害はクレームで判断する
- 見つからなかった検索クエリも記録する(調査の網羅性の証跡になる)
- 断定を避け、「〜の可能性がある」「専門家の確認が必要」という表現を使う
