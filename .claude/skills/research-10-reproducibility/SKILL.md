# 再現性整備(Reproducibility)

第三者(と数ヶ月後の自分)が結果を再現できる状態に、環境・データ・手順を棚卸しして文書化する。

## 前提

開始時に `research/STATUS.md`・`research/experiment-results.md`・`research/implementation-plan.md`・
`research/code-review-log.md`(あれば)を読む。
実際の環境情報はファイルの記憶に頼らず、コマンドを実行して現物から取得する。

## 進め方

### 1. 環境の棚卸し(コマンドで取得)

実環境で以下を実行し、出力を記録する:

- OS・カーネル: `uname -a`, `cat /etc/os-release`
- Python / 処理系: `python --version`
- GPU・ドライバ: `nvidia-smi`(あれば)
- 依存ライブラリ: `pip freeze`(または `uv pip freeze` / `poetry export`)→ バージョン固定ファイルとして保存
- ハードウェア: CPU・メモリ(`lscpu`, `free -h`)

### 2. 実験条件の棚卸し

- 乱数シード: 使用した全シードと、シード固定のコード箇所(Python/NumPy/フレームワーク/CUDA の決定性設定)
- 学習設定・推論設定: 各実験の設定ファイル一覧と、experiment-results.md の結果との対応表
- データセット: 入手方法(URL・ライセンス)、バージョン、前処理手順、分割方法、チェックサム(可能なら)

### 3. 再現手順書

`research/experiment-guide.md` に、クリーンな環境から主要結果を再現するまでの手順を書く:

1. 環境構築(依存インストール)
2. データ取得・前処理
3. 学習実行コマンド(設定ファイル指定込み)
4. 評価実行コマンド
5. 期待される結果(数値と許容ばらつき)

**可能な限り「1 コマンドで主要結果を再現」できるスクリプトを用意する。**

### 4. Docker 方針とチェックリスト

- Dockerfile 作成方針(ベースイメージ、CUDA バージョン、依存の入れ方)。要望があれば Dockerfile 自体も作成する
- 再現性チェックリスト: 環境固定 / シード固定 / データ入手可能 / 設定と結果の対応 / 1コマンド再現 /
  ライセンス確認 / **実験コードの独立レビュー実施(`code-review-log.md` の有無で確認)** の各項目に
  ✅/⬜ を付ける。レビュー未実施の項目があれば、埋めるかリスクとして明記するかをユーザーに確認する

### 5. 出力

```
research/
    reproducibility.md    # 総括とチェックリスト、Docker 方針
    environment.md        # 環境情報(コマンド出力ベース)、依存バージョン固定
    experiment-guide.md   # 再現手順書
```

### 6. 完了処理

- `research/STATUS.md` を更新
- 次のステップ `/research-11-paper-writing` を案内

## 品質基準

- 環境情報は必ず実コマンドの出力から書く(推測・記憶で書かない)
- 「再現できない要素」(独自データ、巨大計算資源など)は隠さず、その旨と代替手段を明記する
